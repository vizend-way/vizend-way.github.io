@FieldSequence 어노테이션 기능 사용

개요​

어노테이션 ‘@FieldSequence’로 시퀀스 관련 코드를 생성할 수 있습니다.

@FieldSequence 어노테이션 추가​

@Getter
@Setter
@NoArgsConstructor
public class Foo extends StageEntity {
    //
    private String name;
    @FieldSequence
    private int firstSequence;
    @FieldSequence
    private int secondSequence;
...

Foo Entity 의 속성을 입력하고 필요한 경우 해당 속성 위에 ‘@FieldSequence’ 어노테이션을 추가하여 관련 코드를 생성할 수 있도록 합니다.

시퀀스 생성 코드​

Backend code 를 생성한 후 FooLogic, FooStore, FooJpaStore 및 FooJpaRepository 에 시퀀스 관련 코드가 함께 생성이 됩니다.

public int nextFirstSequence(String fooId) {
    /* Gen by Vizend Studio v6.1.0 */
    return fooStore.nextFirstSequence(fooId, 1);
}
public int nextSecondSequence(String fooId) {
    /* Gen by Vizend Studio v6.1.0 */
    return fooStore.nextSecondSequence(fooId, 1);
}
public int[] nextFirstSequence(String fooId, int limit) {
    /* Gen by Vizend Studio v6.1.0 */
    int lastSeq = fooStore.nextFirstSequence(fooId, limit);
    int[] sequences = new int[limit];
    int index = 0;
    for (int seq = lastSeq - limit + 1; seq <= lastSeq; seq++) {
        sequences[index] = seq;
        index++;
    }
    return sequences;
}
public int[] nextSecondSequence(String fooId, int limit) {
    /* Gen by Vizend Studio v6.1.0 */
    int lastSeq = fooStore.nextSecondSequence(fooId, limit);
    int[] sequences = new int[limit];
    int index = 0;
    for (int seq = lastSeq - limit + 1; seq <= lastSeq; seq++) {
        sequences[index] = seq;
        index++;
    }
    return sequences;
}
...

FooLogic 클래스에서 자동 생성된 시퀀스 코드입니다.

int nextFirstSequence(String fooId) 메소드는 fooId 에 해당하는 Foo 엔티티의 다음 시퀀스(firstSequence) 값을 반환합니다. 즉 1 만큼 증가한 값을 반환합니다. (예, 만일 해당 시퀀스(firstSequence)가 3 이면 1 을 증가시킨 후 그 값을 반환합니다.)

int nextSecondSequence(String fooId) 메소드는 위와 동일하며 단지 사용하는 시퀀스(secondSequence)가 다릅니다.

int[] nextFirstSequence(String fooId, int limit) 메소드는 fooId 에 해당하는 Foo 엔티티의 다음 시퀀스(firstSequence) 값을 배열로 여러 개 반환합니다. 즉 limit 만큼 증가한 후 시퀀스 초기값과 증가한 값 사이의 값들을 배열로 반환합니다. (예, 만일 ‘limit’이 3 이고 해당 시퀀스(firstSequence)가 3 이면 ‘6’과 같은 배열이 반환됩니다.)

int[] nextSecondSequence(String fooId) 메소드는 위와 동일하며 단지 사용하는 시퀀스(secondSequence)가 다릅니다.

FooStore 인터페이스에서 생성된 시퀀스 코드는 nextFirstSequence(), nextSecondSequence() 메소드입니다.

Int nextFirstSequence(String id, int increment) 는 id 에 해당하는 Foo 엔티티의 firstSequence 를 increment 만큼 증가시킨 값을 리턴합니다.

Int nextSecondSequence(String id, int increment) 는 위와 동일하며 시퀀스만 secondSequence 로 다릅니다.

@Override
public int nextFirstSequence(String id, int increment) {
    /* Gen by Vizend Studio v6.1.0 */
    fooJpaRepository.increaseFirstSequence(id, increment);
    Optional<FooJpo> fooJpo = fooJpaRepository.findById(id);
    if (fooJpo.isEmpty()) {
        throw new NoSuchElementException("Foo id: " + id);
    }
    return fooJpo.get().getFirstSequence();
}
@Override
public int nextSecondSequence(String id, int increment) {
    /* Gen by Vizend Studio v6.1.0 */
    fooJpaRepository.increaseSecondSequence(id, increment);
    Optional<FooJpo> fooJpo = fooJpaRepository.findById(id);
    if (fooJpo.isEmpty()) {
        throw new NoSuchElementException("Foo id: " + id);
    }
    return fooJpo.get().getSecondSequence();
}

엔티티 테이블의 시퀀스 필드를 업데이트하는 쿼리가 JPQL 로 구현되었습니다. 일반적으로 JPQL 은 DML Operation(insert, update, delete)을 지원하지 않습니다. @Modifying 어노테이션을 선언함으로써 DML 이 가능하게 하였습니다.
