Studio 사용 개요

이 문서는 Studio를 처음 접하는 사람들을 위하여 전체적인 사용 방법을 알려주고 빠르게 Drama 개발을 시작하기 위한 문서입니다.

설치​

Menu 에서 Settings… > Plugins > Marketplace 를 선택하고 ‘vizend’로 Plugin 을 검색합니다.

검색된 결과에서 Vizend Studio 의 Install 버튼을 선택하여 설치를 진행합니다. 설치가 되면 IntelliJ IDE를 재시작합니다.

Backend 프로젝트 생성​

File > New > Project… 메뉴를 선택합니다. 나타난 대화창의 왼쪽 메뉴에서 Vizend Studio를 선택합니다.

필요한 옵션을 선택하고 Next 버튼을 선택합니다.

drama-[drama name] 의 규칙으로 Project name을 입력합니다. 그리고 Create 버튼을 선택합니다.

Load Gradle Project 버튼을 선택하여 Gradle build script가 로드될 수 있도록 합니다.

Entity와 Cdo 생성​

domain sub 프로젝트를 선택하고 오른쪽 마우스를 클릭한 후 ‘Create Aggregate entity and Cdo…’를 선택합니다.

나타난 대화창에서 Entity type, Aggregate Name, Entity Name을 각각 입력합니다. Ok 버튼을 누르면 Foo Entity와 FooCdo 클래스가 각각 생성됩니다.

@Getter
@Setter
@NoArgsConstructor
public class Foo extends StageEntity {

    public Foo(String id, ActorKey requesterKey) {
        super(id, requesterKey);
    }

    public Foo(FooCdo fooCdo) {
        super(fooCdo.genId(), fooCdo.getRequesterKey());
        BeanUtils.copyProperties(fooCdo, this);
    }
...
@Getter
@Setter
@Builder
@AllArgsConstructor
@NoArgsConstructor
public class FooCdo extends CreationDataObject {

    @Override
    public String genId() {
        // 
        return super.genId();
    }
...

생성된 코드의 초기 모습입니다.

@Getter
@Setter
@NoArgsConstructor
public class Foo extends StageEntity {
    //
    @FieldImmutable
    private String code;
    private String name;
    private String description;
...

Foo Entity의 속성을 입력합니다. 해당 속성이 수정되지 말아야 할 경우 ‘@FieldImmutable’ 어노테이션을 추가할 수 있습니다.

Entity 속성 입력이 완료되면 에디터의 Context Menu를 이용하여 modifyAttribute() 메소드 내용을 업데이트 할 수 있습니다.

또한 ‘Gen the field enum’을 선택하면 Field의 이름들을 내부 enum 으로 정의할 수 있습니다.

public class FooCdo extends CreationDataObject {
    //
    private String code;
    private String name;
    private String description;

    @Override
    public String genId() {
        //
        return this.code + "_" + this.name;
    }
    ...

한편, Cdo의 속성을 입력합니다. Entity의 Id 규칙을 변경하려면 Cdo 클래스의 genId() 메소드에 규칙을 입력하면 됩니다.

Backend 코드 생성​

Project View에서 Entity를 선택 후 ‘Create Backend Codes…’를 선택합니다.

나타나는 대화창에서 ‘Create All Codes’를 선택하고 모든 코드를 생성합니다.

선택한 Entity와 관련된 Command, Query, Domain, Store 코드가 생성되었습니다.

Feature Code 생성​

Drama에 새로운 Feature 를 추가하려고 할 때 Feature Code 생성 기능을 사용하면 편리하게 기능을 구현할 수 있습니다.

facade sub 프로젝트를 선택하고 ‘Create Feature Request…’를 선택합니다.

만일 무언가를 등록하거나 수정하는 Feature를 구현하려면 Request Type을 CommandRequest로 선택합니다. (조회를 위한 Request Type은 QueryRequest입니다.) Feature Name과 Sub-Feature Name은 여러 Feature Request들을 구분하기 위한 카테고리 역할을 합니다.

package io.vizend.foo.facade.feature.foo.bar.command;

...

@Getter
@Setter
@AllArgsConstructor
@NoArgsConstructor
@AuthorizedRole(FooRole.Director)
public class RegisterActiveBarCommand extends CommandRequest {

    public void validate() {
        /**
         *  FIXME : User Implementation
         * Example.
         * Assert.hasText(skuNo, "'skuNo' is required");
         * Assert.isTrue(increaseRate > 0, "'increaseRate' is greater than 0");
         */
    }
    ...

생성된 FeatureRequest 클래스입니다. package 이름 중 command 앞에 존재하는 foo, bar 는 각각 feature 및 sub-feature 이름입니다. 개발자는 생성된 FeatureRequest에 Request 속성을 추가합니다.

public class RegisterActiveBarCommand extends CommandRequest {
    private String code;
    private String name;
    private String description;

    public void validate() {
        Assert.hasText(code, "'code' is required");
        Assert.hasText(name, "'name' is required");
        Assert.hasText(description, "'description' is required");
    }
    ...

속성을 추가하고 validate() 메소드를 구현합니다.

Project View 화면에서 FeatureRequest를 선택하고 ‘Create Feature Codes…’를 선택합니다.

나타난 대화창에서 Flow Name를 결정합니다. sub-feature 이름을 기준으로 나타나며 다른 Flow Name을 원할 경우 수정합니다.

FlowFacade, FlowResource 그리고 Flow 코드가 생성되었습니다.

@Service
@Transactional
@RequiredArgsConstructor
public class BarFlow {

    public String registerActiveBar(String code, String name, String description) {
        /* FIXME : User Implementation */
        return null;
    }
}

생성된 Flow 클래스의 초기 모습입니다. 개발자는 이 후 Flow 코드를 여기서 추가할 수 있습니다.

Frontend 코드 생성​

지금까지 작업된 backend 코드를 기준으로 프론트 코드를 생성할 수 있습니다.

프로젝트 Root를 선택하고 ‘Create All Frontend Codes…’를 선택합니다.

생성 코드가 위치할 프론트 프로젝트가 아직 생성되지 않았거나 따로 지정되지 않는 경우 이 대화창이 나타납니다. Default 프론트 프로젝트를 원할 경우 ‘Use Default Location’을 체크하고 Ok 버튼을 클릭합니다.

Default 프론트 프로젝트가 아직 생성되지 않아 생성할지 물어봅니다.

나타난 화면 왼쪽 부분에서 생성할 기준이되는 Backend 파일을 선택하고 오른쪽 부분에서 생성 Target을 선택합니다.

프론트 코드가 생성되며 생성 통계를 보여주는 화면이 보여집니다.

Default 위치에서 생성된 drama-foo-front 프로젝트를 따로 열어서 생성된 코드를 확인합니다.
