프로젝트 옵션 변경

Group ID 옵션 변경​

프로젝트 vizend-foo 를 선택하고 -> 오른쪽 마우스를 클릭 -> Vizend Studio -> ‘Change Project Options…’를 클릭합니다.

변경 가능한 화면이 나옵니다. 변경 가능한 옵션은 Store type 과 Drama information 두 가지가 있습니다.

Group ID 를 변경하고 OK 버튼을 클릭합니다.

변경된 부분을 확인하기 위해 Entity 와 Cdo 를 생성합니다.

foo-aggregate 모듈을 선택하고 -> 오른쪽 마우스를 클릭하고 -> Vizend Studio -> ‘Create Aggregate entity and Cdo…’를 클릭합니다.

필요한 정보를 입력하고 OK 버튼을 클릭합니다.

package 이름은 기존 io.vizend 에서 kr.drama 으로 변경이 되었습니다.

Bar클래스를 선택하고 -> 오른쪽 마우스를 클릭하고 -> Vizend Studio -> ‘Create Backend Codes…’를 클릭합니다. (관련 절차 Backend Codes 생성 쳅터에서 있음.)

event, logic, store, doc, repository package 관련 코드가 모두 생성되었습니다.

만약 위에서 Backend 코드를 생성 완료된 상태에서 DB store type 을 변경해야 한다면 Change Project Options 메뉴에서 변경 가능합니다.

프로젝트 vizend-foo 를 선택하고 -> 오른쪽 마우스를 클릭하고 ->Vizend Studio -> ‘Change Project Options…’를 클릭합니다.

기존에 MongoDB를 사용했던 상태에서 MariaDB로 변경하고 'OK' 버튼을 클릭합니다.

DB 를 변경 후 Backend 코드를 다시 생성해야 합니다.

MariaDB 를 기준으로 관련 코드가 생성되었습니다.->jpa,jpo

기존의 dependencies 은 MongoDB 를 기준으로 생성되었기 때문에 MariaDB 로 변경할 때 error 가 발생할 수 있습니다.

이런 경우, dependencies 를 MariaDB 에 맞게 수정해야 합니다. 기존 MongoDB 의 dependencies 입니다.

Error 를 수정하기 위해 ‘build.gradle’ 파일에서 수정해야 합니다.

foo-domain, foo-feature, foo-façade, foo-boot 에 있는 모든 build.gradle 파일을 수정해야 합니다.

build.gradle(foo-domain)파일에서 mongodb ->jpa 로 수정.

build.gradle(foo-feature)파일에서 mongodb ->jpa 로 수정.

build.gradle(foo-facade)파일에서 mongodb ->jpa 로 수정.

build.gradle(foo-boot)파일에서 mongodb ->jpa 로 수정.

build.gradle 파일을 수정 후 Gradle 를 ‘Reload’ 하면 error 도 함께 수정됩니다.
