Entity 수정

Entity 수정​

이전 챕터에서 생성했던 Bar 엔티티로 속성을 추가하고 코드를 수정하는 방법을 설명합니다.

필요한 속성을 Bar 클래스에 입력한 후 속성에 필요한 ValueObject 및 Enum 타입을 정의할 수 있습니다. 이러한 클래스들은 일반적으로 vo 패키지에 저장합니다. Address ValueObject 타입 및 BarStatus Enum 타입을 vo 패키지 내에서 정의합니다.

속성에 몇가지 지정된 어노테이션을 설정하면 코드 생성에 영향을 줄 수 있습니다.

@FieldImmutable 을 지정하면 이 속성 값은 수정이 불가능하다라는 점을 선언하게 됩니다. 따라서 코드 생성시 modifyAttribute 메소드에서 속성을 수정하는 코드가 사라지게 됩니다.

@FieldSourceId 을 지정하면 이 속성은 다른 엔티티의 ID 라는 점을 가리키게 됩니다.

ValueObject 추가 및 수정​

foo-aggregate 에서 aggregate.entity package 를 선택하고 -> New -> Java Class 를 클릭합니다.

vo package 와 Address 클래스를 동시에 생성합니다.

vo package 가 생성되었고 Address 클래스도 함께 생성되었습니다.

Address vo 클래스의 예시 코드 샘플입니다. 개발자가 필요에 따라 수정할 할 수 있습니다.

위 클래스를 만들고 엔티티에서 Address 클래스의 import 를 추가합니다.

마찬가지로 BarStatus Enum도 생성합니다.

foo-aggregate 에서 aggregate.entity package 를 선택하고 -> New -> Java Class 를 클릭합니다.

BarStatus 를 Enum 으로 생성합니다.

vo package 내부에서 BarStatus enum 이 생성되었습니다.

BarStatus vo 클래스의 예시 코드 샘플입니다. 개발자가 필요에 따라 수정할 할 수 있습니다.

위 클래스를 만들고 엔티티에서 BarStatus import 를 추가합니다.

Bar 클래스의 속성이 완료되었습니다. 개발자가 필요에 따라 다른 속성을 수정 및 추가를 할 수 있습니다.

위 속성을 입력 후 Vizend Studio 를 이용하여 modifyAttribute() 메소드 내용을 업데이트 할 수 있습니다.

modifyAttributes 를 선택하고 -> 오른쪽 마우스 클릭하고 -> Vizend Studio -> ‘Update modifyAttbutes()...’ 클릭합니다.

입력된 속성에 따라 modifyAttribute() 메소드가 업데이트되었습니다. 속성이 @FieldImmutable 로 지정되었다면, 해당 속성은 수정이 불가능하므로 modifyAttribute() 메소드에서 해당 속성을 변경하는 코드가 존재하지 않습니다. 속성이 @FieldSourceId 로 지정되었다면, 해당 속성은 수정이 가능하므로 modifyAttribute() 메소드에서 해당 속성을 수정하는 코드가 존재합니다. ValueObject 인 Address 는 JSON 형식으로 전달됩니다.

BarStatus 는 Enum 으로 정의되어 있어, 해당 Enum 의 값은 valueOf() 메소드를 통해 Enum 형태로 전환됩니다.

bar 를 선택하고 -> 오른쪽 마우스 클릭하고 -> Vizend Studio -> ‘Gen the field enum’ 클릭합니다.

엔티티 외부에서 속성명을 문자열로 표현할 때 오타를 방지하기 위해 Filed Enum 을 활용합니다.

예시를 보면, 문자열에 오타가 있어도 오류로 인식하지 않아 식별이 어렵습니다.

BarField 와 같은 Enum 을 사용하면 컴파일 오류를 발생시키므로 오타를 쉽게 감지할 수 있습니다.

BarField 를 사용한 예시 코드입니다. 이를 통해 올바른 필드를 선택하고 오타를 방지할 수 있습니다.

ValueGroup 추가 및 수정​

foo-aggrregaate 에서 vo package 를 선택하고 -> New -> Java Class 를 클릭합니다.

예시로 EmailGroup 클래스에서 ValueGroup 를 구현한 샘플을 만들었습니다. 작성 방법은 ValueObject 와 동일합니다.

ValueGroup 클래스를 생성 후 관련 속성을 Bar 클래스에 추가합니다.
