Field암호화

일반적으로 데이터베이스에 노출에 민감한 내용을 저장할 때 암호화 기술을 사용합니다. 예를 들어 로그인 기능을 사용할 때 필요한 password의 노출을 방지하기 위해 Field 암호화를 사용할 수 있습니다.

‘@FieldEncrypted’ 어노테이션 사용​

Entity 클래스를 생성한 후 로그인 관련 속성들을 입력합니다. 

@Getter
@Setter
@NoArgsConstructor
public class Bar extends StageEntity {
    //
    public enum BarField {
        name, userId, password
    }

    private String name;
    private String userId;
    @FieldEncrypted
    private String password;
    ...

Password 속성은 민감한 정보이고 노출을 방지하기 위해 ‘@FieldEncrypted’ 어노테이션을 추가합니다.

모든 속성과 설정을 마친 후에는 자동화 기능을 이용하여 ‘modifyAttributes()’ 메소드와 필드를 나열하는 내부 Enum을 생성합니다. 그리고 BarCdo 속성도 함께 완성합니다.

‘Gen the field enum’ 기능으로 생성된 내부 Enum

public class Bar extends StageEntity {
    //
    public enum BarField {
        name, userId, password
    }
    ...

‘Update modifyAttributes()’ 기능으로 메소드 코드 생성

@Override
protected void modifyAttributes(NameValueList nameValues) {
    //
    for (NameValue nameValue : nameValues.list()) {
        String value = nameValue.getValue();
        switch(nameValue.getName()) {
            case "name":
                this.name = value;
                break;
            case "userId":
                this.userId = value;
                break;
            case "password":
                this.password = value;
                break;
            default:
                throw new IllegalArgumentException("Update not allowed: " + nameValue);
        }
    }
}

Backend Codes 생성​

위 상태에서 Entity와 관련된 Backend Code를 생성합니다.

Backend Codes생성이 완료된 후에는 BarJpo 클래스에서 @FieldEncrypted 관련 코드를 확인할 수 있습니다.

@Getter
@Setter
@NoArgsConstructor
@Entity
@Table(name = "BAR")
@Generated("Gen by Vizend Studio v6.1.0")
public class BarJpo extends StageEntityJpo {
    /* Gen by Vizend Studio v6.1.0 */
    private String name;
    private String userId;
    private String password;
    public BarJpo(Bar bar) {
        /* Gen by Vizend Studio v6.1.0 */
        super(bar);
        BeanUtils.copyProperties(bar, this);
        this.password = ValueEncryptor.encrypt(bar.getPassword());
    }
    public Bar toDomain() {
        /* Gen by Vizend Studio v6.1.0 */
        Bar bar = new Bar(getId(), genRequesterKey());
        BeanUtils.copyProperties(this, bar);
        bar.setPassword(ValueEncryptor.decrypt(this.password));
        return bar;
    }
    ...

this.password = ValueEncryptor.encrypt(bar.getPassword()); 코드는 사용자가 입력한 password 정보를 암호화하여(encrypt) 다른 사람이 알 수 없는 문자열로 변경한 후 DB에 저장합니다.

반대로, bar.setPassword(ValueEncryptor.decrypt(this.password)); 코드는 DB에 있는 암호화된 password 정보를 읽어와 원래의 password로 복호화 한 후(decrypt) Entity 필드로 세팅합니다.

Feature 생성​

이제 DB 암호화 기능을 테스트 해 보기 위해 몇 가지 기능을 추가하겠습니다. 먼저 Bar Entity를 등록하는 기능입니다.

Façade모듈을 선택하고 Feature Request를 생성합니다.

즉, bar-façade를 선택하고 -> Vizend Studio -> ’Create Feature Request’를 클릭합니다.

FooBarFlow 클래스의 registerBar() 메소드는 실제로 개발자가 구현해야 하는 부분입니다.

아래에 Bar Entity를 등록 할 수 있는 기능을 예로 작성했습니다.

@Service
@Transactional
@RequiredArgsConstructor
public class FooBarFlow {
    //
    private final BarLogic barLogic;

    public String registerBar(String name, String userId, String password) {
        BarCdo barCdo = BarCdo.builder()
                .name(name)
                .userId(userId)
                .password(password)
                .build();

        String id = barLogic.registerBar(barCdo);
        return id;
    }
}
...

필드를 final로 선언하면 Spring이 FooBarFlow 클래스의 생성자를 통해 BarLogic Bean을 해당 필드에 주입합니다.

Spring Boot를 실행하고Swagger-UI를 사용하면 제공되는 API 목록을 확인할 수 있습니다. 방금 추가한 register-bar와 find-bar 기능을 확인할 수 있습니다.

마찬가지로 DB에 저장된 데이터를 조회하면 암호화된 password가 처음에 입력한 문자열로 변환되는지 Swagger-UI에서 확인할 수 있습니다.
