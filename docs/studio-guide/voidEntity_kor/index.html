VoidEntity 사용 방법

개요​

Studio에서 Entity의 Identity를 직접 정의할 수 있는 VoidEntity를 사용할 수 있습니다. 기존의 DomainEntity, StageEntity는 Identity를 플랫폼에서 관리합니다.

사용방법​

Entity를 생성하는 메뉴를 선택하고 표출되는 다이얼로그에서 Entity type을 VoidEntity를 선택합니다.

@Getter
@Setter
@NoArgsConstructor
public class Foo extends VoidEntity {

    public Foo(String id) {
        // TODO : Set the id to your own value
    }

    public Foo(FooCdo fooCdo) {
        super();
        BeanUtils.copyProperties(fooCdo, this);
    }

    @Override
    public String getId() {
        // TODO : Get the id from your own value
        return null;
    }
...

생성된 VoidEntity 코드의 초기 모습입니다. Logic이나 Event 등 다른 플랫폼 코드들과 Identity를 주고 받기 위해서 생성자 내부와 get 메소드의 내부를 개발자가 구현해야 합니다.

@Getter
@Setter
@NoArgsConstructor
public class Foo extends VoidEntity {
    @FieldId
    private String myFooId;
    private String name;

    public Foo(String id) {
        this.myFooId = id;
    }

    public Foo(FooCdo fooCdo) {
        super();
        BeanUtils.copyProperties(fooCdo, this);
    }

    @Override
    public String getId() {
        return myFooId;
    }
...

위 코드에서 myFooId는 엔티티 아이디입니다. myFooId가 아이디임을 Studio에게 알리기 위해 ‘@FieldId’ 어노테이션을 추가합니다. 그리고 생성자와 getId 메소드를 구현합니다.

그리고 나서 Foo 엔티티를 선택하고 모든 다른 코드를 생성합니다. Identity 위치가 Foo 엔티티에 있으므로 생성된 코드는 StageEntity, DomainEntity와 조금씩 다릅니다.

@Getter
@Setter
@NoArgsConstructor
@Entity
@Table(name = "FOO")
public class FooJpo implements JsonSerializable {
    /* Gen by Vizend Studio v5.1.0 */
    @Id
    private String myFooId;
    private String name;

    public FooJpo(Foo foo) {
        /* Gen by Vizend Studio v5.1.0 */
        BeanUtils.copyProperties(foo, this);
    }

    public Foo toDomain() {
        /* Gen by Vizend Studio v5.1.0 */
        Foo foo = new Foo();
        BeanUtils.copyProperties(this, foo);
        return foo;
    }
...

생성된 JPO에서 myFooId에 ‘@Id’ 어노테이션이 적용되었습니다. 그리고 JPO 생성자와 toDomain 메소드 코드는 기존의 DomainEntity의 JPO와 다릅니다.

이상으로 VoidEntity 사용법을 마칩니다.
