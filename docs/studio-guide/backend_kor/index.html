Backend Code 생성

Backend Code 생성​

foo-aggregate 에서 Bar 클래스를 선택하고 -> 오른쪽 마우스 클릭하고 -> Vizend Studio -> ‘Create Backend Codes…’ 클릭합니다.

Bar 클래스에 ValueGroup 속성을 추가 후 ‘Create Backend Codes’ 다시 실행합니다.

필요한 옵션을 선택하고 생성합니다.

‘Create All Codes’를 선택하고 OK 버튼을 클릭합니다.

생성 완료된 화면이 나오면 OK 버튼을 클릭합니다.

기본적으로 aggregate, store 및 façade 모듈에서 관련 코드가 생성됩니다.

Store Type 이 MongoDB 인 경우 store 모듈을 MongoDB 기준으로 코드를 생성합니다. 

@Getter
@Setter
@NoArgsConstructor
@Document(collection = "BAR")
@Generated("Gen by Vizend Studio v6.1.0")
public class BarDoc extends StageEntityDoc {
    /* Gen by Vizend Studio v6.1.0 */
    private String name;
    private String code;
    private Address address;
    private BarStatus barStatus;
    private String emailGroupMainEmail; //
    private String emailGroupSubEmail;
    private String fooId;
    ...

저장 클래스(BarDoc)를 생성할 때 Entity 속성 중 ValueGroup 이 포함 된 경우 ValueGroup 내의 속성으로 분할되어 생성합니다.

이 상태에서 Store Type 을 변경할 수 있습니다. Vizend-foo 를 선택하고 -> 오른쪽 마우스 클릭하고 -> Vizend Studio -> ‘Change Project Options…’ 클릭합니다.

Store Type 는 MariaDB 로 변경하여 OK 버튼을 클릭합니다.

관련된 Backend 코드를 다시 생성합니다.

@Getter
@Setter
@NoArgsConstructor
@Entity
@Table(name = "BAR")
@Generated("Gen by Vizend Studio v6.1.0")
public class BarJpo extends StageEntityJpo {
    /* Gen by Vizend Studio v6.1.0 */
    private String name;
    private String code;
    @Column(columnDefinition = "TEXT")
    private String addressJson; // io.vizend.foo.aggregate.foo.entity.vo.Address
    @Enumerated(EnumType.STRING)
    private BarStatus barStatus;
    private String emailGroupMainEmail; //
    private String emailGroupSubEmail;
    private String fooId;
    ...

MariaDB 기준의 JPA 코드가 생성되었습니다.

MongoDB 와 마찬가지로 Entity 에 ValueGroup 이 포함된 경우 ValueGroup 내부 속성을 풀어서 JPO 를 생성하였습니다.

Entity 속성이 ValueObject 인 경우에는 속성을 JSON 형식으로 변환하여 단일 Column 에 저장할 수 있도록 코드가 생성됩니다.

BarStatus 와 같이 Entity 속성이 Enum 인 경우에는 @Enmuerated 어노테이션이 추가됩니다.

Studio 는 해당 프로젝트의 vo 패키지 내에 존재하는 ValueObjec, ValueGroup, Enum 타입을 식별 할 수 있습니다. 만일 Entity 에 사용된 특정 Enum 타입이 프로젝트 외부에서 만들어진 것이라면 Studio 는 이것이 Enum 인지 식별 할 수 없습니다. 이러한 경우 @FieldEnum 어노테이션을 추가하여 Studio 에게 이 속성이 Enum 타입인지 알려줄 수 있습니다.

Store 는 BarStore 와 BarOptionStore 두 가지 생성되었습니다. BarStore 는 자동으로 생성된 영역이기 때문에 개발자가 수정하거나 추가할 수 없습니다.

만약 개발자가 Store 메소드를 추가해야 한다면, BarOptionStore에추가해야 합니다.

BarOptionStore 에 메소드를 추가한 후 Studio 의 ‘Create Option Store Codes’ 기능을 실행합니다.

OptionStore 클래스를 선택하고 -> Vizend Studio -> ‘Create Opion Store Codes…’를 클릭합니다.

생성에 반영된 코드는 BarLogic, BarOptionJpaStore, 그리고 BarJpaOptionRepository 세 가지 입니다.

public List<Bar> findByEmail(String email) {
    /* Gen by Vizend Studio v6.1.0 */
    return barOptionStore.retrieveByEmail(email);
}
    ...

BarLogic 에 추가된 메소드를 확인하실 수 있습니다.

@Override
public List<Bar> retrieveByEmail(String email) {
    /* Gen by Vizend Studio v6.1.0 */
    return BarJpo.toDomains(barJpaOptionRepository.findByEmail(email));
}
    ...

BarOptionJpaStore 에 구현된 저장로직을 볼 수 있습니다.

@Generated("Gen by Vizend Studio v6.1.0")
public interface BarJpaOptionRepository extends JpaRepository<BarJpo, String> {
    /* Gen by Vizend Studio v6.1.0 */
    List<BarJpo> findByEmail(String email);
}
    ...

BarJpaOptionRepository 에도 마찬가지로 추가된 코드를 확인할 수 있습니다.

만약 또 추가한 내용이 있다면 위 절차대로 다시 진행하여 코드를 생성할 수 있습니다.
