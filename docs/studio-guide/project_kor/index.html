Backend 및 Frontend 프로젝트 생성

Backend 프로젝트 생성​

Menu 에서 File -> New -> Project 를 선택합니다.

대화창의 왼쪽 메뉴에서 Vizend Studio 를 선택하고 Project Style 을 Drama Backend 로 선택합니다. 그리고 나머지 옵션을 선택 한 후 Next 를 선택합니다.

옵션화면은 Project Style, Project template, Store type, Drama information 및 Build Method 5 가지 구분 되어 있습니다.

Project Style 옵션은 원하는 프로젝트에 따라 Backend 또는 Frontend 를 선택할 수 있습니다.

Project template 는 CQRS 하나로 고정되어 있습니다. CQRS 는 Command Query Responsibility Segregation 약자로, 자바 프로그램에서 명령 및 쿼리를 처리하는 책임을 분리하고, 읽기 및 쓰기 작업을 독립적으로 최적화하여 독립적으로 처리하는 패턴입니다. 이벤트를 사용하여 서로 다른 구성 요소 간의 응용 프로그램 상태를 동기화하는 것을 포함합니다.

Add Mashup module 을 체크하면 모듈 중에 Mashup 모듈을 함께 생성합니다.

Skip Event Generation 을 체크하면 이벤트 관련 코드가 생성되지 않습니다.

Store type 은 MongoDB, MariaDB 및 Oracle 과 같은 다양한 데이터베이스 타입을 선택할 수 있습니다.

Drama Information 항목에는 Group ID 와 Module Prefix 를 입력할 수 있습니다. Group ID 는 생성하려는 드라마의 Group 을 결정하며 패키지 이름을 결정합니다. Module Prefix 로 프로젝트 모듈 이름을 변경할 수 있습니다.

Build Method 는 Gradle 만 선택 가능합니다.

위 옵션들을 선택 후 Next 버튼을 클릭합니다.

프로젝트 이름과 저장위치 정보를 입력합니다.

Project name 입력시 Drama 네이밍 규칙에 따라 소문자를 기본으로 하고 단어 사이를 ‘-’로 구분해야 합니다. (특수문자는 사용을 못합니다) Module Prefix 가 없다면 ‘-‘ 뒤에 입력한 단어는 프로젝트 모듈 이름으로 사용됩니다.

‘-‘를 사용하지 않고 한 단어만으로 입력해도 됩니다.

관련 정보를 입력 후 Create 버튼을 클릭합니다.

프로젝트 이름과 Module Prefix 입력 여부에 따라 모듈이름 표기가 다를 수 있습니다.

기본적을 모듈 이름은 프로젝트 이름을 기반으로 표기됩니다. 즉 프로젝트 이름에서 ‘-‘ 뒤에 입력된 단어로 표기됩니다. (‘-‘를 입력하지 않는 경우, 한 단어만으로 표기됩니다.)

Module Prefix 를 입력한 경우 모듈 이름을 자유롭게 결정할 수 있습니다.

프로젝트 이름에 ‘-‘를 하나 이상 입력한 경우, ‘-‘ 뒤의 모든 이름이 모듈이름으로 표기됩니다.

Load Gradle Project 버튼을 클릭하여 Gradle build script 가 로드될 수 있도록 합니다.

만약 Gradle Load 가 실패한다면, SDK 버전을 21 로 변경하여야 합니다.

Menu 에서 File -> Project Structure -> Project -> SDK 순서로 선택합니다. SDK 옵션에서 21 버전을 찾아 하여 변경합니다.

만약 21 버전이 없다면 Download 할 수 있습니다. Version 및 Vender 옵션은 21 버전으로 선택한 후 Download 합니다.

21 버전 다운로드 후 선택하여 OK 버튼을 클릭합니다.

마찬가지로 Gradle JVM 도 21 버전으로 변경해야 합니다.

Menu 에서 File -> Settings -> Build -> Gradle 순서로 선택합니다.

설정을 모두 21 버전으로 선택하였다면 OK 버튼을 클릭합니다.

‘Reload’버튼을 다시 클릭하여 재실행합니다.

Reload 가 성공하였습니다.

Backend 생성 파일​

build.gradle 파일

build.gradle파일에는 사용하는 모듈버전 및 dependency와 관련된 정보가 포합됩니다.

BootApplication 파일 boot 모듈 아래를 살펴보면 BootApplication이 생성됩니다.

MongoTransactionConfig 파일 Mongodb 를 선택한 경우, MongoTransactionConfig도 생성됩니다.

schema 파일 스키마 생성을 위한 script 파일도 함께 생성됩니다. 이를 통해 DB 를 설정해야 합니다.

application.yml 파일 boot 실행에 필요한 application.yml 파일도 생성됩니다.

Role 파일 facade 모듈에서 Role 클래스 하나가 생성됩니다. 이 클래스는 드라마 Role 을 정의하는 용도로 사용됩니다. 개발자는 여기서 드라마 Role 을 정의할 수 있습니다.

Frontend 프로젝트 생성​

Menu 에서 File -> New -> Project -> 대화창의 왼쪽 메뉴에서 Vizend Studio 를 선택하고 -> Frontend 프로젝트를 선택하여 생성합니다. 필요한 옵션들을 선택 후 Next 버튼을 클릭합니다.

Frontend 프로젝트명은 Backend 프로젝트명 뒤에 ‘-front‘를 함께 붙여야 합니다. 왜냐하면 Studio 는 프론트 코드 생성시 Backend 코드를 기준으로 생성하는데 Studio 가 생성된 프론트 코드를 프론트 프로젝트에 넣어주어야 합니다. 이 때 기본적으로 ‘[Backend 프로젝트 명]-front’ 프로젝트를 찾습니다.

관련 정보를 입력 후 Create 버튼을 클릭합니다.

Frontend 프로젝트를 생성됩니다.

Frontend 생성 파일​

editorconfig 파일 텍스트 편집기 설정 파일로, 코드 스타일을 통일하기 위한 설정을 담고 있습니다.

eslintignore 파일 ESLint 에서 디렉토리를 지정하는 파일입니다.

eslintrc 파일 ESLint 설정 파일로, JavaScript 코드의 문법 및 스타일을 정의합니다.

gitignore 파일 Git 버전 관리에서 제외할 파일 목록을 지정하는 파일입니다.

gitlab-ci.yml 파일 GitLab CI/CD 파이프라인을 정의하는 파일입니다. 코드의 빌드, 테스트, 배포 등을 자동화하는 데 사용됩니다.

npmrc 파일 npm 설정 파일로, npm 패키지 관리자의 동작을 제어하는 설정을 담고 있습니다.

prettierignore 파일 Prettier 에서 디렉토리를 지정하는 파일입니다.

prettierrc 파일 Prettier 설정 파일로, 코드의 서식을 정의하는 설정을 담고 있습니다.

package.json 파일 package.json 파일은 Frontend 개발에 필요한 환경 및 모듈 정보를 담고 있습니다.

README.md 파일 프로젝트에 대한 설명을 담고 있는 마크다운 형식의 파일로, 주로 프로젝트의 사용법, 설치 방법, 기능 소개 등을 기록합니다.

tsconfig.json 파일 tsconfig.json 파일은 typescript 컴파일을 하기 위한 옵션 정보를 담고 있습니다.

turbo.json 파일 Turbo 프레임워크에서 사용하는 설정 파일로, 프로젝트의 빌드, 실행, 환경 설정 등에 대한 정보를 담고 있습니다.

Backend 프로젝트에서 Frontend 프로젝트도 생생할 수 있습니다. 프로젝트 vizend-foo 를 선택하고 오른쪽 마우스를 클릭한 후 ‘Change Front Project...’를 클릭합니다.

Frontend 프로젝트가 이미 존재한다면 Use Default Location 을 선택하고 OK 버튼을 클릭합니다.

만약 Frontend 프로젝트가 존재하지 않고 다른 지정한 Frontend 프로젝트가 있다면 Use Default Location 을 선택 해제하고 다른 Frontend 프로젝트를 선택합니다.

다른 Frontend 프로젝트를 선택하고 Open 버튼을 클릭합니다.

지정한 파일 선택 후 OK 버튼을 클릭합니다.

만약 Frontend 프로젝트가 존재하지 않고 Use Default Location 을 선택한 경우, OK 버튼을 클릭하면 프론트 프로젝트를 생성할지 여부를 묻습니다.

Yes 버튼 클릭하면 생성하게 됩니다.

OK 버튼을 클릭하면 해당 위치에 Frontend 프로젝트가 생성됩니다.
