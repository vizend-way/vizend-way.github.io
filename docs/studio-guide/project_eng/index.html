Create Backend & Frontend Project

Create Backend Project​

Menu bar -> File -> New -> Project -> Select ‘Vizend Studio’ from the left menu of the dialogue box -> choose Drama Backend for Project Style -> After selecting the remaining options, click Next button.

The options are divided into five sections: Project Style, Project Template, Store Type, Drama Information, and Build Method.

Choose one of the project style from options based on your need.

CQRS has been configured within the project template. CQRS stands for Command Query Responsibility Segregation, is a design pattern that separates the responsibilities of handling commands and queries in a Java program, optimizing read and write operations independently for separate handling. This includes synchronizing the application state between different components by using events.

If you check the’ Add Mashup module’ option, the Mashup module will be created along with other modules.

If you check the ‘Skip Event Generation’ option, event code will not be generated.

You can choose from a variety of databases such as MongoDB, MariaDB, and Oracle.

In the Drama Information section, you can input the Group ID and Module Prefix. The Group ID determines the group of the drama and the package’s name. You can change the project module name with the Module Prefix.

Only Gradle is available for the Build Method option. After selecting all the options, click Next button.

Enter the project name and choose the project location in the following window.

According to the Drama naming rule, words should be lowercase letters and separated by hyphen '-'. (Special characters are not allowed.) If there is no Module Prefix, the word entered after '-' will be used as the project module name.

You can also enter a single word without using '-'. After entering the Project namd and location, click Create button.

The directory name and module name may vary based on the project name and prefix you entered.

The default module name is based on the project name. In other words, it will be represented by the word entered after '-' in the project name. (If '-' is not entered, it will be represented as a single word.)

The module name is based on the additional prefix input.

"If '-' is entered more than once, all words following each '-' will be considered part of the module name.

Click “Load Gradle Project” button on the bottom right of the project to allow the Gradle build script to load.

If you can not Load Gradle successfully, go to Project Structure to change the version of SDK version and Settings to change the Gradle JVM version.

Menu bar -> File -> Project Structure -> Project -> SDK Find the version of SDK 21 from options.

If there’s no SDK21, click Download JDK.

Choose the Version and Vendor to Version 21, and click Download button.

Back to the Project Structure, choose the version21 and click OK button.

The Gradle JVM version should also be changed to version 21.

Menu bar -> File -> Settings -> Build -> Gradle

Choose Gradle JVM to version21 and Click OK button.

After setting up the SDK version 21, click the “Reload” button.

Reload completed successfully.

Generated Files for Backend​

build.gradle file

The build.gradle file contains information related to the module versions and dependencies used.

BootApplication file Under the boot module, the BootApplication is also generated.

MongoTransactionConfig file The MongoTransactionConfig is also generated if you choose Mongodb as your database.

schema file A script file for schema generation is also generated. The database needs to be configured based on the script file.

application.yml file The application.yml file required for boot execution is also generated.

Role file In the facade module, a ‘Role’ class is generated, which is used to define drama roles. If additional drama roles are needed, developers can add them to this class.

Create Frontend Project​

Menu bar -> File -> New -> Project -> Select ‘Vizend Studio’ from the left menu of the dialogue box -> choose Drama Frontend for Project Style.

Select the options you need and click Next button.

The frontend project name should be appended with '-front' following the backend project name. This is because the Studio generates frontend code based on the backend code, and the Studio needs to place the generated frontend code into the frontend project. By default, it looks for a project named '[Backend project name]-front'.

After entering the Project namd and location, click Create button.

The Frontend Project has been generated.

Generated Files for Frontend​

editorconfig file .editorconfig file is a text editor configuration file used for maintaining consistent coding styles across different editors and IDEs.

eslintignore file .eslintignore file is for specifying directories in ESLint.

eslintrc file .eslintrc file defines the syntax and style of JavaScript code.

gitignore file .gitignore file specifies the list of files to be excluded from Git version management.

gitlab-ci.yml file GitLab CI/CD pipeline configuration file used to define automated build, test, and deployment pipelines for your project.

npmrc file npm configuration file used to configure npm behavior and settings, which control the operation for npm package administrator.

prettierignore file This is the file used to specify directories for Prettier.

prettierrc file This is the Prettier configuration file, containing settings that define the formatting of the code.

package.json file The package.json file contains information about the environment and modules required for Frontend development.

README.md file Markdown file containing documentation for your project, including usage instructions, installation guidelines, project overview, and any other relevant information.

tsconfig.json file The tsconfig.json file contains options information for TypeScript compilation.

turbo.json file Configuration file used by Turbo framework, if applicable, to specify project settings like build configurations, environment variables, and runtime options.

You can also create a Frontend Project within a Backend Project. Project vizend-foo -> quick right click -> Vizend Studio -> Click 'Change Front Project...'

If a Frontend project already exists, select 'Use Default Location' and click OK button.

If a Frontend project does not exist and another specified Frontend project exists, uncheck 'Use Default Location' and select the other Frontend project.

Select the other Frontend project and click the Open button.

After selecting the specified file, click OK button.

If a Frontend project does not exist and 'Use Default Location' is selected, click OK button will pop up the window asking whether to create a Frontend project.

Click Yes button to create Default Front Project.

Click OK button, you can find the Frontend Project at the specified location you choose.
