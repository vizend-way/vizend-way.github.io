Overview of using Studio

This document is intended to provide an overview of how to use Studio for those who are new to Studio, and to get started with Drama development quickly.

Install​

Search “Vizend Studio” from the following menu and install.

Menu bar -> File -> Settings -> Plugins -> Marketplace -> vizend search In the search results, select the "Install" button for Vizend Studio to proceed with the installation. Once the installation is complete, restart the IntelliJ IDE.

Create a Backend Project​

Menu bar -> File -> New -> Project -> Select "Vizend Studio" from the menu on the left-hand side of the dialog box Select the options you need and click Next button.

To manually set the module name for your project, enter the name in the "Module Prefix" field. If left blank, the module name will be automatically generated.

Select the "Project Location" and follow the Drama naming rules (for example, use lowercase and hyphens(-) only, avoid special characters) when entering the project name in the "Project name" field. Then click Create button.

The entered Project name is used to create the project module name.

If you do not enter a Module Prefix, this is how the created project will appear.

If you enter 'demo' as the Module Prefix, this is how the project will appear.

Click “Load Gradle Project” button on the bottom right of the project to allow the Gradle build script to load.

If you can not Load Gradle successfully, go to Project Structure to change the version of SDK version and Settings to change the Gradle JVM version to 21.

Menu bar -> File -> Project Structure -> Project -> SDK Find the version of SDK 21, if there’s no SDK21, click Download JDK

Choose the Version and Vendor to Version 21, and click Download button.

Back to the Project Structure, choose the version21 and click OK button.

Menu bar -> File -> Settings -> Build -> Gradle Choose Gradle JVM to version21 and Click OK button.

After setting up the SDK version 21, click “Load Gradle Project” button to reload.

Create Entites and Cdos​

Select the Project tree from the left menu -> foo-aggregate -> quick right click -> Vizend Studio -> Click ‘Create Aggregate entity and Cdo…’

Choose the Entity type, enter Aggregate Name, and Entity Name in the dialog box, then click OK button.

Bar Entity and BarCdo Class will be created in individually.

Here's a snippet of the generated code for Bar class.

@Setter

public class Bar extends StageEntity {

    }

    public Foo(BarCdo barCdo) {
        super(barCdo.genId(), barCdo.getRequesterKey());
        BeanUtils.copyProperties(barCdo, this);
    }

Enter the attributes for Entity Class.

You can ADD ‘@FieldImmutable’ annotation if the attributes are not modified.

You can also create 'Gen the field enum' to define the names of the fields as an internal enum.

After entering the attributes, scroll down the entity class to find modifyAttributes() and update the contents of the modifyAttributes() by using the editor’s Context Menu.

@Override
protected void modifyAttributes(NameValueList nameValues) {
//
    for (NameValue nameValue : nameValues.list()) {
        String value = nameValue.getValue();
        switch(nameValue.getName()) {
            case "name":
                this.name = value;
                break;
            case "address":
                this.address = value;
                break;
            default:
                throw new IllegalArgumentException("Update not
                        allowed: " + nameValue);
        }
    }
}
    ...

Contents of modifyAttributes method after updated.

Cdo is a class for creating domain Entities. To create an Entity, the attributes must be defined.

public class BarCdo extends CreationDataObject {
    //
    private String name;
    private String address;
    private int age;

    @Override
    public String genId() {
        //
        return this.code + "_" + this.name;
    }
    ...

After entering the attributes of Cdo, define the rules for Entity Id by using the genId() method.

public static BarCdo sample() {
//
    return BarCdo.builder()
            .name("Dave")
            .address("Seoul, South Korea")
            .build();
}
    ...

You can define a sample() method for developers who are unfamiliar with how to use Cdo.

Based on the attributes entered in the BarCdo class, you can define the sample() method by using the builder pattern.

Create Backend Codes​

Back to the Project Tree -> Select Bar Entity -> quick right click -> Vizend Studio -> Click ‘Create Backend Codes…’

Select “Create All Codes” in the dialog box and click OK button to generate all backend codes.

The dialog box displaying the creation result shows the number of generated files and the status of the created codes.

The 'Command', 'Query', 'Domain', and 'Store' codes within the selected Entity have been generated as shown in the following Project Tree.

If additional methods are needed beyond those provided by the automatically generated Store interface, you can use the "OptionStore interface". The naming of additional methods can follow the Derived Query Methods of Spring Data.

Create Feature Codes​

It’s also EASY to create a NEW Feature code to your Drama project.

Select the foo-facade from Project Tree-> quick right click -> Vizend Studio -> Click ‘Create Feature Request…’

Select Request Type in the following dialog box(CommandRequest for Registers or Modifies, QueryRequest for Queries).

Enter Feature Name / Sub-Feature Name to distinguish the difference in categories.

Click OK button to generate Feature Codes.

package io.vizend.foo.facade.feature.foo.bar.command;

...

@Getter
@Setter
@AllArgsConstructor
@NoArgsConstructor
@AuthorizedRole(FooRole.Director)
public class RegisterActiveBarCommand extends CommandRequest {

    public void validate() {
        /**
         *  FIXME : User Implementation
         * Example.
         * Assert.hasText(skuNo, "'skuNo' is required");
         * Assert.isTrue(increaseRate > 0, "'increaseRate' is greater than 0");
         */
    }
    ...

This is the generated FeatureRequest class. The 'foo' and 'bar' preceding the package name 'command' represent the feature and sub-feature names, respectively.

Developers can add Request attributes to the generated FeatureRequest if needed.

public class RegisterActiveBarCommand extends CommandRequest {
    //
    private String name;
    private String address;
    private int age;

    public void validate() {
        /**
         *  FIXME : User Implementation
         * Example.
         * Assert.hasText(skuNo, "'skuNo' is required");
         * Assert.isTrue(increaseRate > 0, "'increaseRate' is greater than 0");
         */
        Assert.hasText(name, "'name' is required.");
        Assert.hasText(address, "'address' is required");
        Assert.isTrue(age >= 18, "'age' must over 18");
    }
    ...

Enter the feature attributes and revise the content of the validate() method if necessary.

Back to the Project tree ->Select the RegisterActiveBarCommand (CommandRequest Class) -> quick right click -> Vizend Studio -> Click ‘Create Feature Codes…’

The Flow Name is basically the one you enter the Feature name and Flow/Seek Logic Name from the previous process. Flow Name can be modified if you need a different one.

Click OK button to create Flow Class.

The ‘FlowFacade’, ‘FlowResource’ and ‘Flow’ codes have been generated.

Create Frontend Codes​

Now Generate the Frontend Code based on the backend code you’ve worked on so far.

Back to the Project Tree -> Select the vizend-foo -> quick right click -> Vizend Studio -> Click ‘Create All Frontend Codes…’

This dialog indicates that the frontend project location has not been created or specified yet. If you want to use the default location from the project, check the 'Use Default Location' box and click OK button.

Click Yes button to create Frontend Project.

Frontend Project has been created and click OK button to continue.

Select the backend files as the basis for creation on the left side of the dialog box, and choose the creation target on the right side of the dialog box.

The dialog box shows the result of generated codes, click OK button to view generated codes.

Open the “drama-foo-front project” in new window from the Default location to view the generated frontend codes.
