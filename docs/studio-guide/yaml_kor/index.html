Entity, Cdo 생성을 위한 YAML 작성 가이드

개요​

Vizend Studio는 YAML 포맷의 파일로 Entity 및 Cdo를 생성하는 기능이 있습니다. 이 문서는 이 때 적용 가능한 YAML 문서 작성 방법을 설명합니다.

기본 입력​

groupId: io.vizend
appName: small-talk

groupId 와 appName은 코드 생성시 패키지명 결정에 관여합니다. 필수로 입력합니다.

Entity 정보 입력​

entities:
  - name: RoomBoard
    type: Stage
    aggregate: room

entities 아래 단계의 목록으로 입력합니다.

name과 type을 입력합니다. type의 종류는 Stage, Domain, Data, Void 가 있습니다.

aggregate 이름을 입력합니다. aggregate는 필수 입력 정보입니다. 생성시 Entity 패키지 결정에 관여합니다.

Attribute 정보 입력​

Attribute 정보 입력 방법은 Entity 뿐만 아니라 ValueObject, ValueGroup 에도 동일하게 적용됩니다.

Attribute 입력 개요​

entities:
  - name: Address
    type: Stage
    aggregate: address
    attributes:
      - name: city
        type: String
        comment: 도시
        example: Seoul

attributes 아래 단계의 목록으로 입력합니다.

name을 입력합니다. 입력하지 않으면 type명을 참조하여 유추하여 attribute가 생성됩니다.

type은 primitive 형 또는 java 타입 명을 입력합니다. 입력하지 않으면 String으로 간주합니다.

comment를 입력하는 경우 필드 주석이 생성됩니다.

example을 입력하는 경우 sample() 메소드 코드 생성에 사용됩니다.

Attribute 입력 방법​

1. name 만 입력​

attributes:
  - name: question

name 속성만 지정할 수 있습니다. 이 경우 String 타입의 question 속성을 생성합니다.

private String question;

2. name과 type 입력​

attributes:
  - name: time
    type: long

name과 type을 지정하는 방법이 가장 일반적입니다. type은 primitive 타입 뿐만 아니라 Java Class 이름도 가능합니다.

private long time;

3. list 속성 사용​

attributes:
  - name: usages
    list: io.vizend.accent.domain.xray.usage.FeatureUsage

만일 속성 타입이 List 이라면 Generic Type을 정의할 수 있습니다. 위의 경우 아래로 코드가 생성됩니다. 만일 name 이 생략된다면 list에 정의한 타입명으로 속성 이름이 결정됩니다.

private List<FeatureUsage> usages;

4. immutable​

attributes:
  - name: time
    type: long
    immutable: true

immutable 속성을 true로 설정하면 생성되는 엔티티 속성에 ‘@FieldImmutable’ 어노테이션이 추가되어 수정 불가능한 속성을 생성합니다.

@FieldImmutable
private long time;

Entity 관계 정보 입력​

Entity 간의 관계 설정에 따라 Entity 코드가 생성됩니다. oneToMany, manyToOne, oneToOne 세 가지 속성을 사용하여 1:N 및 1:1 관계를 정의할 수 있습니다.

일대다(1:N) 관계 정의​

정의한 두 Entity간의 관계가 일대다 관계가 있을 경우 oneToMany, manyToOne 속성을 사용할 수 있습니다.

다음 예제는 RoomBoard와 RoomieBoard가 1:N 관계일 경우입니다. One측 Entity 설정

entities:
  - name: RoomBoard
    ...
    oneToMany:
      entity: RoomieBoard

Many측 Entity 설정

  - name: RoomieBoard
    ...
    manyToOne:
      entity: RoomBoard

위의 설정에서 생성된 Entity Code입니다. One측 Entity Code

public class RoomBoard extends StageEntity {

    ...
    private transient List<RoomieBoard> roomieBoards;
    
    ...

Many측 Entity Code

public class RoomieBoard extends StageEntity {

    ...
    @FieldSourceId
    private String roomBoardId;
    
    ...

위 코드에서 One측 Entity는 Many측 Entity 객체를 개념적으로 List 형태로 가지고 있을 수 있습니다. 그러나 실제로는 객체를 가지지 않으므로 해당 필드를 transient 로 생성합니다.

Many측 Entity 객체에서 One측 참조자를 가질 수 있으므로 FieldSourceId 유형의 필드를 생성합니다.

일대일(1:1) 관계 정의​

두 Entity 간의 관계가 일대일일 경우 oneToOne 속성을 사용하여 관계정의를 할 수 있습니다. 이때 owing 하위 속성으로 둘 중 하나의 Entity에서 소유관계를 선언할 수 있습니다.

One(Owning)측 Entity 설정

    - name: Roomie
      ...
    oneToOne:
      entity: RoomieBoard
      owning: true

One(Not Owning)측 Entity 설정

      - name: RoomieBoard
        ...
      oneToOne:
        entity: Roomie

생성된 Entity Code입니다.

One(Owning)측 Entity Code

public class Roomie extends StageEntity {

    ...
    private transient RoomieBoard roomieBoard;
    
    ...

One(Not Owning)측 Entity Code

public class RoomieBoard extends StageEntity {

    ...
    @FieldSourceId
    private String roomieId;
    
    ...

위 코드에서 소유자 Entity 측에서는 상대방 Entity 객체를 개념적으로 소유하고 실제로는 소유하지 않으므로 transient 필드로 선언되었습니다.

또한 소유하지 않는 상대방 Entity에서는 소유자의 Entity Id를 FieldSourceId 형태로 가지고 있습니다.

Value Object 정보 입력​

Entity를 정의하며 필요한 ValueObject, ValueGroup 및 Enum 정보를 입력할 수 있습니다. 3가지 모두 Entity의 Attribute 형태로 정의하거나 Entity 외부에서 정의할 수 있습니다.

ValueObject 또는 ValueGroup 정의​

Entity 내부 정의

     entities:
     - name: RoomBoard
       type: Stage
       aggregate: room
       attributes:
           ...
             - valueObject:
             name: LastTalk
             attributes:
               - name: roomId
               - name: name
               - name: message

Entity 내부에서 정의하는 방법은 Entity의 attribute 정의시 같이 정의하는 방법입니다. attributes 하위 리스트 항목으로 valueObject 또는 valueGroup 을 정의하고 이름과 속성을 정의합니다. 이 때 aggregate는 따로 입력하지 않으며 상위 Entity에 정의된 aggregate에 생성됩니다.

Entity 외부 정의

     valueObjects:
     - name: LastTalk
       aggregate: room
       attributes:
         - name: roomId
         - name: name
         - name: message

ValueObject/ValueGroup을 Entity 외부에 정의할 수 있습니다. valueObjects 또는 valueGroups 하위 리스트 항목으로 정의할 수 있습니다. 이 때 주의할 점은 aggregate를 반드시 정의해야 합니다. 정의한 ValueObject 또는 ValueGroup은 해당 aggregate에 생성됩니다.

ValueObject Code

public class LastTalk implements ValueObject {
    private String roomId;
    private String name;
    private String message;
    ...

Enum 정의​

Entity에서 사용하는 Enum 타입이 있을 경우 정의할 수 있습니다. ValueObject와 마찬가지로 Entity 내부 또는 외부에서 정의할 수 있습니다.

Entity 내부 정의

     entities:
     - name: Roomie
       type: Stage
       aggregate: room
       attributes:
         - enumType:
             name: RoomRole
             entries:
               - Owner
               - Admin
               - Roomie

Entity의 attributes 하위 리스트 항목으로 입력 가능합니다. entries 에서 타입들을 나열할 수 있습니다. aggregate 는 상위 Entity를 따릅니다.

Entity 외부 정의

     enumTypes:
     - name: RoomRole
       aggregate: room
       entires:
         - Owner
         - Admin
         - Roomie

enumTypes 하위 리스트 항목으로 입력 가능합니다. 이 경우 aggregate를 반드시 입력해야 합니다.

Enum Type Code

public enum RoomRole {
    Owner, Admin, Roomie
}

생성됨 Enum 코드는 위와 같습니다.
