Service User

이 문서는 Checkpoint를 이용하여 Microservices 연동에 필요한 API 키를 발급받고 해당 키를 이용해 서비스를 호출할 수 있는 방법을 설명합니다.

개요​

Vizend에서 사용자는 아래와 같이 3가지 유형의 사용자로 구분합니다.

  • Citizen User
    Vizend 플랫폼 위에서 일반적인 사용자를 가르킵니다. Metro를 통해 등록되고 작업 공간에 할당된 사용자가 Drama Role을 가지고 사용하게 됩니다. 주로 사용자 UI를 통해 접근하는 일반 사용자를 위해 사용됩니다. Checkoint는 Authentication만 담당하고 Drama 별로 개별 Authorization을 가지고 있습니다.
  • Internal User
    도메인의 BC(Bounded Context) 내에서 서버 간의 제약없는 호출을 위한 연계 계정입니다. 서비스 간의 인터페이스를 위한 부분으로 서비스명으로 구분됩니다. 외부에서는 절대 호출하지 못하고 prologue 라이브러리를 통해 자동으로 적용됩니다. Checkpoint 서비스에서 사용자 정보 데이터를 생성하지 않고 규약 만으로 사용됩니다.
  • Service User
    도메인 BC(Bounded Context) 외부에서 타 플랫폼 및 서비스 등과 같이 앱을 사용하지는 않지만 Vizend 플랫폼과 API 방식으로 통해 연계할 수 있습니다. Open API 인증 방식이고 인증을 위한 Username, Password가 있고 그것을 통해 API 인증 Token을 발급받아 사용합니다. 서비스 사용자는 텍스트 기반의 Role을 가질 수 있고 이를 이용해 개별 Drama에서 API 접근을 제어하게 됩니다.

서비스 사용자 관리 API​

서비스 사용자 관리를 위한 다음과 같은 API가 제공됩니다. 모든 API는 Internal User로 접근할 수 있고 일반 Citizen User, Service User 등은 호출해 사용할 수 없습니다.

서비스 롤​

서비스 Role을 관리하는 화면입니다. Vizend 플랫폼에서 제공할 각각의 서비스 역할을 등록하고 관리할 수 있습니다.

서비스 롤 목록 조회​

서비스 롤 등록​

서비스 롤이 등록되면 상태(active)는 기본으로 활성화 됩니다.

POST: /restful/service-users/service-roles

Arguments JSON

NameTypeExampleDescription
code (*)String"mon-viewer"서비스 롤의 코드를 지정합니다.
Drama에서 이 이름으로 서비스 호출 권한을 식별합니다.
name (*)String"Monitoring Viewer"서비스 롤의 이름을 지정합니다.
descriptionString"Monitoring viewer user role"서비스 롤 설명을 지정합니다.

Response

등록된 서비스 롤의 아이디를 반환합니다.

a595073e-d4a3-49b3-b829-6bee4249ceae

서비스 롤 수정​

PATCH: /restful/service-users/service-roles/{service-role-id}

Arguments JSON

NameTypeExampleDescription
nameString"Monitoring Viewer"서비스 롤의 이름을 변경합니다.
activebooleantrue서비스 롤의 활성화 상태를 변경합니다.
descriptionString"Monitoring viewer user role"서비스 롤 설명을 변경합니다.

Response

수정된 서비스 롤의 아이디를 반환합니다.

a595073e-d4a3-49b3-b829-6bee4249ceae
