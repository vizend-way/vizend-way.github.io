SSO Login

이 문서는 Checkpoint를 이용하여 SSO User를 등록하고, SSO Login을 이용하여 토큰을 발급받는 방법을 설명합니다.

개요​

SSO 인증을 이용하는 경우 사용자 인증은 SSO 단계에서 완료되었다고 가정하고, checkpoint 에서는 username 만을 이용하여 토큰을 발급받게 됩니다. User 등록 시 비밀번호는 TinyUUID로 임의 설정되며(일반 로그인 방지), 로그인 시에는 암호화된 username으로 Token 발급을 요청합니다.

Required Versions​

Backend

accentVersion = '6.1.21'
prologueVersion = '6.1.33'
checkpointVersion = '6.1.27'

Frontend

{
  "dependencies": {
    "@vizendjs/accent": "6.1.20",
    "@vizendjs/checkpoint-stub": "6.1.10"
  }
}

SSO 사용자 등록 API​

사용자 등록을 위한 API와 클라이언트입니다.

POST: /restful/citizen-users?sso=true
BODY

{
  "pavilionId": "NT:1",
  "username": "clienttest",
  "email": "clienttest@text.com",
  "displayName": "Client Test",
  "additionalInformation": {}
}

RESPONSE
등록된 Citizen의 id를 반환합니다.

{
  "id": "2@NT:1"
}

CLIENT

import io.vizend.checkpoint.client.citizenuser.CitizenUserClient;
import io.vizend.checkpoint.client.citizenuser.sdo.RegisterSsoCitizenUserRequest;

public String registerSsoCitizenUser(String pavilionId, String username, String email, String displayName, Map<String, Object> additionalInformation) {
  // 
  RegisterSsoCitizenUserRequest registerSsoCitizenUserRequest = RegisterSsoCitizenUserRequest.builder()
          .pavilionId(pavilionId)
          .username(username)
          .email(email)
          .displayName(displayName)
          .additionalInformation(additionalInformation)
          .build();
  return citizenUserClient.registerSsoCitizenUser(registerSsoCitizenUserRequest);
}

3. SSO Login (Backend)​

Backend 에서 CitizenUserClient 로 로그인을 하는 경우 Public Key를 이용하여 username을 암호화한 후 CitizenUserClient 를 이용하여 token 을 획득합니다.

application.yaml
publicKey와 Basic token 을 세팅합니다.

Basic token은 vizend.prologue.internal-auth.client의 id와 secret을 조합한 문자열의 Base64 인코딩 문자열을 사용합니다.
ex) id: nextree, secret:nextree1234 인 경우
nextree:nextree1234 인코딩 ⇒ bmV4dHJlZTpuZXh0cmVlMTIzNA==

vizend:
  checkpoint:
    client:
      rest:
        base-url: http://localhost:9002
        request-timeout-seconds: 600
client-test:
  sso:
    token:
      dml6ZW5kOnZpemVuZHNlY3JldA==
    public-key: |
      MFwwDQYJKoZIhvcNAQEBBQADSwAwSAJBAM1yji8ppN9BwqhwMtqm
      JX7w+NDCoO3J90FlbbV6mmWPmHbsNRdnd8vg9hG6ySjNtPLIH97P
      MVhixFeTnlhpHVMCAwEAAQ==

accent의 RsaEncryptor를 이용하여 username을 암호화하여 CitizenUserClient의 ssoLogin 메서드를 통해 token을 획득합니다.


package io.vizend.test.feature.checkpoint.flow;

import io.vizend.accent.util.crypto.RsaEncryptor;
import io.vizend.checkpoint.client.citizenuser.CitizenUserClient;

public class CheckpointFlow {
    //
    private final CitizenUserClient citizenUserClient; // 
    @Value("${client-test.sso.public-key}")
    private String publicKey;
    @Value("${client-test.sso.token}")
    private String ssoToken;

    public String ssoLogin(String pavilionId, String username) {
        // 
        String encryptedUsername;
        try {
            encryptedUsername = RsaEncryptor.encrypt(username, publicKey.replaceAll("\\s", ""));
        } catch (Exception e) {
            throw new RuntimeException(e);
        }
        log.info("encryptedUsername: {}", encryptedUsername);
        SsoAuthenticationToken ssoAuthenticationToken = citizenUserClient.ssoLogin(pavilionId, encryptedUsername, ssoToken);
        log.info("token ==> {}", ssoAuthenticationToken.toPrettyJson());
        return ssoAuthenticationToken.getAccessToken();
    }
}

4. SSO Login (Frontend)​

Front의 경우 Dock의 ssoLogin hooks를 이용합니다. PrivateKey의 경우 env로 설정할 경우 아래와 같이 설정합니다.

env 변수 선언
/src/vite-env.d.ts

/// <reference types="vite/client" />
interface ImportMetaEnv {
  // 다른 환경 변수들에 대한 타입 정의...
  readonly VITE_SSO_PUBLIC_KEY: string
}

interface ImportMeta {
  readonly env: ImportMetaEnv
}

env value 설정
KeyPair에서 획득한 Public Key를 설정합니다. /.env

VITE_SSO_PUBLIC_KEY=MFwwDQYJKoZIhvcNAQEBBQADSwAwSAJBAM1yji8ppN9BwqhwMtqmJX7w+NDCoO3J90FlbbV6mmWPmHbsNRdnd8vg9hG6ySjNtPLIH97PMVhixFeTnlhpHVMCAwEAAQ==

실제 로그인은 useAuth()의 ssoLogin() 함수를 이용합니다.
username의 경우 RsaEncryptor를 이용하여 암호화합니다.

import { useAuth } from './useAuth';
import { RsaEncryptor } from '@vizendjs/accent';

const SsoLogin = () => {
    //
    const { ssoLogin } = useAuth();
    
    const handleLogin = async () => {
        const {username, pavilionId} = loginForm;

        if (!username || !pavilionId) {
            alert('Not enough auth information.');
            return;
        }

        await ssoLogin({
            encryptedUsername: RsaEncryptor.encrypt(username, import.meta.env.VITE_SSO_PUBLIC_KEY),
            pavilionId,
        }).catch((e) => handleError(e));
    };
}
